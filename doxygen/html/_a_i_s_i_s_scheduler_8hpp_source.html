25 source.json (extern)
26  - In the same folder of AISISScheduler.hpp, there should be a file
27  named source.json. This shall comprise of a list of courses
28  available from AISIS. The repository shall provide a sample
29  source.json for reference.
30 class Course
31  - comprises of a course, a section, and other details regarding it.
32  Data comprising ::Course shall be taken from a JSON. More about it
33  later.
34 std::list <std::string> requiredSubjects
35 std::list <AISISScheduler::Course> theList
36 std::list <AISISScheduler::Course> resultList
37 bool scheduleTable
38  - This is a schedule table: 6x28. 6 because there are 6 (working) days
39  in a week (0 being Mon, 5 being Sat), and 28 because there are 28 possible
40  30-minute slots (from 07:00 to 21:00). Each cell can either be 0 or 1: 0 if
41  that time slot has not been taken by any subject, and 1 if it already has
42  been. This shall be memset'd every time the program shall create a new
43  viable schedule. More on that later.
44 
45 Functions:
46 
47 void extractUserInput()
48  - the user shall input a list of courses they are required to take.
49  The user shall be asked to put the name of the course, and hit Enter
50  after each time they type something. Once the user is done, they may
51  type Enter without placing anything. The list of courses shall be
52  pushed onto ::requiredSubjects.
53 void pushToTheList()
54  - Once ::requiredSubjects is filled up, this function shall be used
55  to read from source.json, check ::requiredSubjects, create the
56  necessary ::Course's and push them onto ::theList. The source.json
57  may take up a chunk of space, but, once the function exits, the
58  JSON shall be freed up from memory.
59 bool isCompatible(AISISScheduler::Course subj)
60  - This shall determine whether subj is compatible with ::scheduleTable
61  by checking subj.weekDays and subj.timeSlot, creating a temporary time
62  table akin to ::scheduleTale, and checking whether there are any conflicts.
63 */
